MySQL schemas are structured to maintain referential integrity across users, products, and orders.

<summary><b>⬡ &nbsp;Nova Capital — Stock Market Portfolio Management System</b> &nbsp;<code>Jun 2025 – Oct 2025</code></summary>

<br/>

A full-stack web application that simulates real stock trading with portfolio tracking, buy/sell transaction modules, and a personalized investor dashboard. Built with secure authentication, live balance management, and a clean MySQL schema to handle concurrent transaction and holding data across 6 major company stocks.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | HTML, CSS, JavaScript, Node.js, MySQL |
| **Scale** | 6 major company stocks · full transaction history · multi-user portfolio isolation |
| **Performance** | Dynamic balance updates on every transaction · real-time dashboard reflecting current holdings |
| **Security** | Password encryption · session management · CSRF protection on all state-changing operations |
| **Impact** | Demonstrates production-grade web app security practices and relational DB design for financial data |
| **Repository** | [![GitHub](https://img.shields.io/badge/Nova--Capital-FFC107?style=flat-square&logo=github&logoColor=000000&labelColor=1a1a1a)](https://github.com/NamburiVivek) |

</div>

The database is structured around three normalized tables — users, transactions, and stock holdings — with foreign key constraints ensuring data consistency across buy/sell operations. The transaction module performs atomic balance deduction and holding updates within a single DB transaction, preventing partial state writes. The dashboard aggregates holdings and account balance in a single optimized query for fast page loads.

</details>

<details>
<summary><b>⬡ &nbsp;Airline Reservation System — Java Desktop Application</b> &nbsp;<code>Jan 2025 – Apr 2025</code></summary>

<br/>

A Java-based desktop application for managing airline ticket reservations end-to-end. Features a fully interactive GUI built with Java Swing covering flight search, seat booking, cancellation, and payment processing — designed using core OOP principles and a modular class architecture with UML documentation.

<div align="center">

| Attribute | Detail |
|-----------|--------|
| **Stack** | Java, Java Swing, ArrayList (in-memory storage) |
| **Scale** | Multi-flight management · customer records · booking and cancellation lifecycle |
| **Performance** | Responsive Swing GUI · efficient in-memory ArrayList operations for flight and customer data |
| **Security** | Input validation across all booking and payment flows · structured error handling |
| **Impact** | Demonstrates mastery of OOP design patterns, GUI development, and system architecture with UML class diagrams |
| **Repository** | [![GitHub](https://img.shields.io/badge/Airline--Reservation--System-FFD700?style=flat-square&logo=github&logoColor=000000&labelColor=1a1a1a)](https://github.com/NamburiVivek/Airline-Reservation-System) |

</div>

The system architecture is designed around four core OOP pillars: Encapsulation (private flight/customer data with getter/setter access), Inheritance (base Reservation class extended by domestic and international variants), Abstraction (interface-driven booking contracts), and Polymorphism (overridden payment processing for different ticket classes). UML class diagrams document the full entity relationship model, and enums handle flight status and seat class constants cleanly.
