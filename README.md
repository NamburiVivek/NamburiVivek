<br/>

Deployed across three Fortune 500 clients, processing 2M+ queries per month with 99.97% uptime SLA.

<div align="center">

| Attribute | Detail |
|-----------|--------|

| **Repository** | [![GitHub](https://img.shields.io/badge/neuralsearch--platform-7C3AED?style=flat-square&logo=github&logoColor=white)](https://github.com/arjunsharma-dev/neuralsearch-platform) |

</div>

The platform implements a multi-stage retrieval architecture: an ANN index (HNSW via pgvector) handles initial dense retrieval, a BM25 sparse layer covers lexical recall, and a cross-encoder re-ranker (fine-tuned on domain data) produces the final ranked set. Context assembly is handled by an adaptive chunker that respects document structure, and a guardrails layer performs hallucination detection via semantic entailment scoring before response delivery.

<br/>

</details>

Handles 800K+ task executions per day with exactly-once delivery semantics and sub-second scheduling latency.

<div align="center">

| Attribute | Detail |

| **Performance** | Sub-100ms scheduling latency · exactly-once delivery · zero message loss over 18 months |
| **Security** | mTLS between all workers · audit logging · role-based pipeline access |
| **Impact** | Replaced 3 fragmented legacy systems · 40% infrastructure cost reduction · 5× throughput increase |
| **Repository** | [![GitHub](https://img.shields.io/badge/meridian--orchestrator-6D28D9?style=flat-square&logo=github&logoColor=white)](https://github.com/arjunsharma-dev/meridian-orchestrator) |

</div>

Worker nodes communicate over gRPC streaming channels, with a Raft-based consensus module ensuring leader election and task assignment durability. The observability stack exposes 200+ custom Prometheus metrics with pre-built Grafana dashboards for SRE consumption.
